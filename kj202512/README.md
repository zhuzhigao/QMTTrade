# 多策略分仓系统（kj202512）

> 四个风格迥异的子策略并行运行，共用一个 QMT 账号，通过独立账本实现持仓隔离，总预算 8 万元。

---

## 策略总述

本策略是聚宽平台经典"多策略分子账户并行"框架的 xtquant 移植版，将 8 万元资金拆分为四个独立子策略，每个子策略持有不同风格的标的，通过低相关性实现组合层面的平滑收益。

**整体资金分配：**

| 子策略 | 文件 | 资金 | 占比 | 核心逻辑 |
|--------|------|------|------|----------|
| ETF 轮动 | `kj202512_etf.py` | 16,000 元 | 20% | 动量 + RSRS 择时，4 只 ETF 选 1 |
| PB 低估值 | `kj202512_pb.py` | 16,000 元 | 20% | PB < 0.98 且 ROE 高，全 A 选 1 |
| 小市值多因子 | `kj202512_xsz.py` | 24,000 元 | 30% | 三组质量/动量因子，小市值优先，选 3 |
| 菜场大妈 | `kj202512_dama.py` | 24,000 元 | 30% | 高股息 + 低价 + 小市值，选 1 |

**持仓隔离：** 每个子策略拥有独立的 `StrategyLedger`（`XX_holdings.json`），只操作自己账本中的持仓，四个进程互不干扰。

---

### 子策略详述

#### 1. ETF 轮动策略（16,000 元）

从 4 只 ETF（上证180、创业板100、纳指100、黄金ETF）中选出动量最强的 1 只持有。

**选股流程：**
1. 对每只 ETF 计算 25 日线性回归动量分数：`年化收益率 × R²`
2. 减去 200 日长期动量 / 6（均值回归惩罚，防止追高）
3. 若最高分与最低分之差不在 `(0.1, 15)` 区间，说明各 ETF 趋势趋同或极度分化，本次空仓
4. RSRS 择时过滤：对通过分数的 ETF，用 18 日高低价回归斜率与历史 250 日分布对比，当前斜率需 > 均值 - 2σ 才允许买入

**执行时间：** 每日 09:35（若目标 ETF 未变化则不操作）

---

#### 2. PB 低估值策略（16,000 元）

从全 A 股中寻找资产极度折价、盈利能力又强的"隐形好公司"。

**选股流程：**
1. 基础过滤：排除科创板、北交所、ST、停牌
2. 财务过滤（同时满足）：
   - PB（市净率）< 0.98：资产打折出售
   - ROE > 15%：高效利用股东资本（xtquant 无直接 ROA 字段，以 ROE 近似）
   - EPS > 0：正盈利
   - 净利润 > 0 且同比正增长：排除业绩恶化
3. 按 ROE 降序排列，取前 3 只候选，持仓 1 只
4. 涨跌停过滤（当日涨停/跌停的不操作）

**执行时间：** 每月首个满足条件的交易日 09:35

---

#### 3. 小市值多因子策略（24,000 元）

原版使用聚宽专属 `get_factor_values()` API，xtquant 移植版改用财务数据构建三组近似因子，核心思路不变：用多因子模型筛出高质量小盘股。

**选股流程：**
1. 基础过滤：排除科创/北交/ST/停牌，过滤上市不足 400 天的次新股
2. 获取全A股财务数据（PershareIndex / Income / Balance）
3. 构建三组因子，各取全市场前 10%：
   - **组1（质量）**：ROE × 0.5 + 净利润率 × 0.3 + EPS × 0.2，要求 EPS > 0
   - **组2（动量/盈利）**：净利润增速 × 0.7 - 资产负债率 × 0.3，要求 EPS > 0
   - **组3（低负债质量）**：-D/A × 0.6 + 净利润率 × 0.4，要求 EPS > 0
4. 取三组并集，按总市值升序（小市值优先）
5. 取前 5 只候选，持仓 3 只
6. 涨跌停过滤

**风控：** 4 月空仓（清仓静待）；个股跌幅 ≥ 20% 触发止损，静默 28 天

**执行时间：** 每周一 09:35 调仓；每日 14:45 止损巡检

---

#### 4. 菜场大妈策略（24,000 元）

模拟散户偏好：买高分红、低价格、小市值的股票，博取估值修复收益。

**选股流程：**
1. 基础过滤：排除科创/北交/ST/停牌
2. 计算股息率 = DPS（每股现金股息）/ 当前股价，取全市场前 25%（高股息）
3. 按总市值升序排列（小市值优先）
4. 价格 < 9 元过滤（已持仓股豁免，避免频繁换仓）
5. 取前 3 只候选，持仓 1 只
6. 涨跌停过滤

**风控：** 个股跌幅 ≥ 20% 触发止损，静默 28 天

**执行时间：** 每月第 15 个交易日之后首次 10:30 调仓；每日 14:45 止损巡检

## 策略优势、劣势与适用市场环境

### 优势
- **低相关性分散**：四个子策略分别代表技术趋势（ETF轮动）、价值（PB）、质量成长（小市值）、红利（大妈），相互对冲，组合波动低于任何单一子策略
- **多层风控**：每个子策略独立止损，且有涨停打开卖出、空仓月等保护机制
- **完全隔离**：四个进程独立运行，互不干扰，任一子策略停止不影响其他
- **适应性广**：覆盖不同市场风格——ETF 轮动可切换到黄金/纳指做防御，PB 在熊市中也能筛到折价资产

### 劣势 / 风险
- **PB 策略条件极严**：PB < 0.98 且 ROE > 15% 在 A 股同时满足的股票极少，可能长期空仓；ROE 并非 ROA，有偏差
- **大妈策略价格滤网粗糙**：9 元硬阈值无法适应不同市值规模的股票，低价股可能质地差
- **小市值因子暴露**：小市值策略在监管趋严、壳价值下降的环境下容易跑输
- **财务数据滞后**：xtquant 财务数据以公告日为准，季报/年报披露前存在一定盲区
- **全A股扫描耗时**：PB、小市值、大妈策略需对全A股批量获取财务数据，每次调仓可能耗时数分钟

### 适合的市场环境
- **最佳**：市场分化明显（价值股与成长股轮动）、有稳定分红文化、中小盘活跃
- **较好**：震荡市（ETF 轮动切防御，其他子策略价值支撑）
- **不适合**：流动性危机时小市值股票跌幅大且难以卖出；PB 策略在市场情绪高涨时可能完全空仓

| `RSRS_N` | ETF轮动 | 18 日 | RSRS回归窗口 | 标准值，不建议调整 |
| `RSRS_M` | ETF轮动 | 250 日 | RSRS标准化基准 | 标准值，不建议调整 |

| `MAX_HOLD` | 小市值 | 3 | 最大持股数 | 3-5 均合理 |
| `MAX_HOLD` | 大妈 | 1 | 最大持股数 | 可调至 2 |
| `NEW_DAYS` | 小市值 | 400 天 | 次新股过滤门槛 | 升高→更成熟的公司 |
| `TOP_PCT` | 小市值 | 10% | 每组因子取前N% | 调小→更精选；过小会导致候选不足 |
| `HIGH_DIV_PCT` | 大妈 | 25% | 高股息分位线 | 调小→股息率更高但候选更少 |
| `MAX_PRICE` | 大妈 | 9 元 | 价格上限 | 根据市场均价水平调整 |
| `STOPLOSS_LEVEL` | PB/小市值/大妈 | 20% | 止损触发跌幅 | 调小→保护更强但换仓更频繁 |
| `STOPLOSS_SILENCE_DAYS` | PB/小市值/大妈 | 28 天 | 止损后静默天数 | 约等于 20 个交易日 |

## 运行方式

四个子策略**各自作为独立进程运行**，互不依赖：

```bash
# 调试模式（完整跑逻辑，不发真实报单）
python kj202512_etf.py
python kj202512_pb.py
python kj202512_xsz.py
python kj202512_dama.py

# 实盘模式（谨慎！确认逻辑正确后分别启动）
python kj202512_etf.py   -m REAL
python kj202512_pb.py    -m REAL
python kj202512_xsz.py   -m REAL
python kj202512_dama.py  -m REAL
```

**注意**：运行前需确认每个文件中的 `qmt_path` 和 `account_id` 与实际环境一致。
